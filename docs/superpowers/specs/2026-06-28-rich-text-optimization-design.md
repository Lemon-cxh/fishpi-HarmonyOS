# 文章详情富文本优化设计文档

**日期：** 2026-06-28  
**状态：** 待审核

## 背景

文章详情页面的富文本显示存在以下问题：
1. 不支持列表标签（`<ul><li>` 和 `<ol><li>`）
2. 段落之间间距过大
3. 标题字号过大
4. 图片不支持点击查看大图

## 需求确认

### 1. 列表标签支持
- **需求：** 支持无序列表（`<ul><li>`）和有序列表（`<ol><li>`）
- **样式：** 标准列表样式
  - 无序列表：使用圆点符号（•），每项缩进显示
  - 有序列表：使用数字序号（1. 2. 3.），每项缩进显示

### 2. 段落间距优化
- **需求：** 调整段落之间的间距
- **目标间距：** 适中间距（12-16vp）

### 3. 标题字号调整
- **需求：** 减小标题字号
- **目标字号：**
  - H1: 1.5倍 (24px)
  - H2: 1.3倍 (20.8px)
  - H3: 1.15倍 (18.4px)
  - H4-H6: 1.05倍 (16.8px)

### 4. 图片点击查看
- **需求：** 支持点击图片查看大图
- **交互方式：** 全屏弹窗查看
  - 点击图片后弹出全屏弹窗
  - 显示大图
  - 点击空白处或返回键关闭

## 技术方案

### 方案选择
采用**渐进式优化方案**，保持现有架构，最小化改动风险。

### 实现细节

#### 1. 列表标签支持

**数据模型扩展（Models.ets）：**
```typescript
export class RichTextContentModel {
  // 现有字段...
  
  // 列表相关字段
  listType: 'none' | 'ul' | 'ol' = 'none';  // 列表类型
  listItems: string[] = [];  // 列表项内容
}
```

**HTML解析器扩展（HtmlParser.ets）：**
- 添加列表标签解析逻辑
- 识别 `<ul>` 和 `<ol>` 标签
- 提取 `<li>` 标签内容
- 创建列表数据模型

**渲染组件扩展（RichTextView.ets）：**
- 添加列表渲染逻辑
- 无序列表：使用圆点符号（•）+ 缩进
- 有序列表：使用数字序号 + 缩进

#### 2. 段落间距优化

**当前问题：**
- 段落结束后添加 `\n` 换行符
- 导致段落间距过大

**优化方案：**
- 移除段落结束后的 `\n` 换行符
- 使用 `Column` 的 `space` 属性控制间距
- 设置 `space: 12` 实现适中间距

#### 3. 标题字号调整

**当前字号倍数：**
- H1: 1.75倍 → 调整为 1.5倍
- H2: 1.5倍 → 调整为 1.3倍
- H3: 1.25倍 → 调整为 1.15倍
- H4-H6: 1.1倍 → 调整为 1.05倍

**实现位置：** `HtmlParser.ets` 中的标题标签处理逻辑

#### 4. 图片点击查看

**新增组件：ImagePreviewDialog.ets**
```typescript
@Component
export struct ImagePreviewDialog {
  @Prop imageUrl: string = '';
  @Link isShow: boolean;
  
  // 全屏显示图片
  // 支持手势缩放
  // 点击空白处关闭
}
```

**RichTextView.ets 修改：**
- 为 Image 组件添加 onClick 事件
- 点击时显示 ImagePreviewDialog
- 传递图片 URL
